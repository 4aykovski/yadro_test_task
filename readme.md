# YADRO Test Task
Требуется написать прототип системы, которая следит за работой компьютерного
клуба, обрабатывает события и подсчитывает выручку за день и время занятости
каждого стола.
Решение может быть реализовано на Golang.

Решением задания будет: файл или несколько файлов с исходным кодом программы
на языке Golang (версия 1.19 и старше) с использованием go modules, инструкции по
запуску и тестовые примеры (количество тестов – на усмотрение разработчика).
Входные данные представляют собой текстовый файл. Файл указывается первым
аргументом при запуске программы. Пример запуска программы:

$ task.exe test_file.txt

Программа должна запускатьcя в Linux или Windows с использованием docker
container-a (требуется написание Dockerfile). Требуется использование стандартной
библиотеки (https://pkg.go.dev/std). Использование любых сторонних библиотек, кроме
стандартной, запрещено. В решении, кроме файлов с исходным кодом, требуется
предоставить инструкции по запуску программы для проверки.

# Запуск без добавления новых тест-кейсов

1. Клонировать репозиторий
```
    git clone https://github.com/4aykovski/yadro_test_task.git
```
2.  Перейти в директорию проекта
```
    cd yadro_test_task
```
3. Забилдить образ
```
    docker build -t your_image_name:your_tag .
```
4. Запустить контейнер с подготовленными тест-кейсами
```
    docker run -t your_image_name:your_tag
```

# Запуск с добавлением новых тест-кейсов
1. Клонировать репозиторий
```
    git clone https://github.com/4aykovski/yadro_test_task.git
```
2.  Перейти в директорию проекта
```
    cd yadro_test_task
```
3. Создать в папке `cases` новый тест-кейс и заполнить его
```
    nano ./cases/new_test_case.txt
```
4. По желанию добавить в Makefile новый тест-кейс по аналогии с существующими для запуска всех тест-кейсов разом
```
    nano ./Makefile
```
5. Забилдить образ
```
    docker build -t your_image_name:your_tag .
```
6. Запустить контейнер с новым тест-кейсом
```
    docker run -t your_image_name:your_tag ./app ./cases/new_test_case.txt
```
7. В случае добавления тест-кейса в Makefile - запустить все тест-кейсы
```
    docker run -t your_image_name:your_tag
```
